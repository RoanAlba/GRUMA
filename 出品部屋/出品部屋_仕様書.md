# 出品部屋（ListingRoom） 仕様書（正式版）

出品部屋（ListingRoom）は、  
ExhibitionRoom で整えられた作品を  
“市場へ送り出すための儀式の部屋” である。

このファイルは **機能・UI・動作・フロー** を定義し、  
内部構造（出品部屋_構造.md）と矛盾しないように設計されている。

---

# 1. 目的
- 作品を MarketUI に送り出すための **最終ステップ**  
- 作品の情報を確認するだけの **読み取り専用UI**  
- ExhibitionRoom の静けさを保ちつつ、  
  市場へ渡る “空気の切り替え” を行う  
- 出品という行為を **儀式として成立させる**

---

# 2. 出品部屋の世界観（UI側の定義）
- 白磁・灰磁・真鍮の 3 素材で統一  
- 影は極薄  
- アニメーションは最小限（フェードのみ）  
- 音は基本無音  
- MarketUI の UI を持ち込まない  
- 数値・市場性・価格などは表示しない  

→ **“静けさと儀式性” が UI の最優先要件。**

---

# 3. UI構造（機能としての定義）

## ■ 3-1. 作品プレビュー（中央）
- 透明台座の上に作品が静止  
- 回転なし  
- 拡大・縮小なし  
- 影は極薄  
- 動画の場合は静かにループ再生（音なし）

---

## ■ 3-2. 出品情報（右側）
- 読み取り専用  
- 編集UIは存在しない  
- 表示項目（最小構成）  
  - タイトル  
  - 説明文  
  - タグ  
  - 出品形式  
  - 注意書き  
- すべて ExhibitionRoom の空気を壊さない静かなUI

---

## ■ 3-3. 出品ボタン（右下）
- 真鍮の円形ボタン  
- 直径：32px  
- ホバー演出なし  
- 押した瞬間にゲートが開く  
- ボタンは “儀式の開始スイッチ” として扱う

---

# 4. 出品フロー（正式版）

## ■ 4-1. ExhibitionRoom → 出品部屋
- 作品が完成し、展示室で整えられた状態で遷移  
- 0.25秒フェードで空気が切り替わる  
- 作品は台座に静止した状態で表示される

---

## ■ 4-2. 最終確認
- 右側の情報を確認するだけ  
- 編集は不可  
- 作品の位置・光・影は固定

---

## ■ 4-3. 出品（儀式）
- 出品ボタンを押す  
- 出品ゲートが 0.3秒以内で開く  
- 作品が光の中へ吸い込まれる  
- 音はほぼなし  
- ゲートが閉じると “出品完了”

---

## ■ 4-4. 完了後の遷移
- 0.2秒の白磁光フェード  
- MarketUI の該当カテゴリへ遷移  
- 出品された作品が MarketUI に現れる

---

# 5. 禁止事項（UI仕様としての制約）
- 市場性％・売れ行き・価格などの数値表示  
- 派手なアニメーション  
- カラーライト  
- 作品の回転・揺れ  
- 編集UI  
- MarketUI の UI を流用すること  
- ExhibitionRoom の世界観を壊す装飾  

---

# 6. このファイルの役割
- 出品部屋の **機能・UI・動作** を定義する正式仕様  
- 内部構造（出品部屋_構造.md）と組み合わせて  
  “空間＋UI＋儀式” の全体像を完成させる  
- MarketUI との接続を明確化する基準ファイル
