# UIテンプレート（素材判定統合版）

このテンプレートは、リファインルームで作った素材を
UI部品（つまみ・スライダー・パネル）の形状に自動反映させるための
「素材→形ルール」を組み込んだ構造ファイル。

---

## 1. テンプレートの基本構造

UI部品はすべて「枠（固定）」と「素材スロット（差し替え）」で構成する。

- 枠（frame） → 形の骨格。固定。
- 素材スロット（fill / base / knob） → 素材を貼る部分。

素材スロットにリファイン素材を入れると、
素材の性質・光・密度に応じて形が自動で決まる。

---

## 2. 素材判定ルールの適用範囲

### ● つまみ（Knob）
- 性質 → 角の丸さ（丸形／角丸／直線）
- 光 → 影の強さ（なし／薄い／広い）
- 密度 → 厚み（薄い／標準／厚い）

### ● スライダー（Slider）
- 性質 → レール形状（直線／角丸）
- 光 → レール影（なし／薄い）
- 密度 → レール厚み（細い／標準）

### ● パネル（Panel）
- 性質 → 角の丸さ（角丸／直線）
- 光 → 影の有無（なし／薄い）
- 密度 → 枠の太さ（細い／標準）

---

## 3. 素材→形の自動変換ロジック

### ● 性質（柔らかい／硬い／透明／粒子）
- 柔らかい → 角丸・丸形
- 硬い → 直線・薄い形
- 透明 → 細い縁・円形
- 粒子 → 小さめ円形

### ● 光の強さ（強／中／弱）
- 強 → 影なし
- 中 → 薄い影
- 弱 → 広がる影

### ● 密度（高／中／低）
- 高 → 薄い・シャープ
- 中 → 標準
- 低 → 厚みがある・柔らかい

## 5. 素材を入れたときの処理フロー

1. 素材を読み込む  
2. 性質・光・密度を判定  
3. UI部品ごとに形状パラメータを決定  
4. テンプレートの枠にパラメータを適用  
5. 素材スロットに素材を貼る  
6. 完成したUIパーツをセットとして出力

---

## 6. このテンプレートの目的

- 素材を作るだけで UI が完成する  
- デフォルト形状を固定しない  
- 素材の雰囲気が UI にそのまま反映される  
- セット販売に最適化された構造になる
