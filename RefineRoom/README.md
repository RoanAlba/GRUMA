# RefineRoom（リファインルーム）

RefineRoom は、GRUMA の UI を「磨き上げる」ための部屋。
光・影・質感・操作感・動き・空気層など、UI の完成度を高める工程を扱う。

ここでは素材制作やアイデア出しは行わず、
既にある UI を「整える・精製する・静かに仕上げる」ことだけに集中する。

---

## ■ RefineRoom の役割
- UI の質感・光・影の調整
- パーツの造形・動きの磨き込み
- 世界観（アンティーク思想）と UI の接続を管理する
- UI の“静けさ”と“精密さ”を保つ

---

## ■ UI パーツ一覧

### ● スライダーつまみ（UI仕様／antique 接続）
- つまみは「宝石＋アンティーク金属枠」で構成された小さなジュエリー
- UI の操作部品でありながら、アンティーク思想に基づく宝飾物として扱う

■ デフォルト（初期値）
- 形：ラウンド
- カット：カボション
- 色：淡い宝石（UI を邪魔しない初期値）
※ デフォルトは初期値であり、変化後の宝石の形・色・カットは完全に自由。

■ 変化後
- ラウンド／オーバル／ペアシェイプ／クッション／マーキスなど自由
- カットも自由（カボション・ファセット・アンティークカットなど）
- 色も自由（濃色・透明・異物色など）

■ 自動変換と保存
- セッティング台に置くとアンティーク思想に従ってジュエリーへ自動変換
- 完成したジュエリーは保存され、ジュエリールームに飾られる

## ■ UI アニメーション仕様（RefineRoom）

RefineRoom の UI は「静けさ」「精密さ」「工場の空気層」を壊さないため、
アニメーションは最小限・低速・滑らかさを基準とする。

### ● スライダーつまみ（宝石）の動き
- 重さのある宝石を前提にした、わずかに粘性のあるスライド
- 加速・減速は弱く、一定速度に近い
- 位置が変わると中央プレビューの光・影が静かに反応
- 離した瞬間、宝石内部の光がわずかに揺れる

### ● レバー（UI 呼び出し）の動き
- 白金属の機構が静かに展開する動き
- 無音
- 展開速度は遅め
- 戻すときは “吸い込まれるように” 静かに閉じる

### ● パネル（左：素材パネル）の展開
- スライドではなく “淡い透明層が静かに現れる”
- フェードイン 0.2〜0.3 秒
- 影を落とさない

### ● プレビュー（中央）の変化
- 瞬間切り替えはしない
- 0.15 秒の淡いフェードで静かに置き換わる
- 光の反射がわずかに揺れる

### ● 調整レバー（右）の動き
- 精密機械の微調整のような直線的な動き
- 反応は即時
- 影はほとんど落とさない
- プレビューが静かに追従する

### ● UI 全体のアニメーション原則
- 速すぎる動きは禁止
- バウンド・弾性・跳ね返りは禁止
- 影が揺れすぎる動きは禁止
- すべての動きは “工場の空気層を壊さない” を最優先とする

## ■ UI 特殊効果（Effects）

RefineRoom の UI は、光・影・内部反応・空気層の揺れなど、
“静かな工場” の世界観を壊さない範囲で最小限の特殊効果を持つ。

---

### ● 宝石つまみの内部光
- つまみを動かすと、宝石内部の光がわずかに揺れる
- 揺れは 0.1〜0.2 秒で収束する
- 色は現在の宝石色に依存し、過剰な発光はしない
- アンティークジュエリー特有の “内部の鈍い光” を再現する

---

### ● プレビューの光の反応
- つまみの位置に応じて、中央プレビューの光が静かに変化する
- 反応速度は 0.1 秒以内（遅延を感じさせない）
- 光の強さは変化するが、影は揺れすぎない
- 工場の空気層を壊さないため、反応は控えめ

---

### ● レバー展開時の空気層の揺れ
- レバーを引くと、周囲の空気層が “わずかに揺れる”
- 揺れは視覚的にはほぼ見えないが、透明層の屈折で感じられる程度
- 揺れは 0.2 秒以内に収束する
- 工場の静けさを壊さないため、音は一切ない

---

### ● パネル出現時の透明層の変化
- パネルが現れるとき、透明層が薄く重なるようにフェードする
- 透明度は 0.0 → 0.6 程度まで
- 影は落とさない（空気層を守る）

---

### ● UI 全体の特殊効果ルール
- 過剰な光・影・揺れは禁止
- 発光は “宝石内部の鈍い光” のみ許可
- 空気層の揺れは最小限
- すべての効果は “静けさ” を最優先とする

## ■ UI の色ルール（Color Rules）

RefineRoom の UI は、白金属・透明層・宝石の 3 要素で構成されるため、
色は極めて限定的に扱う。色は世界観を壊しやすいため、以下の原則に従う。

---

### ● 白金属（UI の基礎色）
- 基本色は「冷たい白金属色」
- 色味は持たない（青み・黄みを避ける）
- 影は薄く、反射は弱い
- UI の“静けさ”を保つため、白金属は常に無彩色

---

### ● 透明層（パネル・空気層）
- 色は持たない（透明度のみ変化）
- 透明度は 0.0〜0.6 の範囲
- 影を落とさない
- 工場の空気層を壊さないため、色の濁りは禁止

---

### ● 宝石（つまみ）
- 色は唯一の“彩度を持つ要素”
- デフォルトは淡い宝石色（UI を邪魔しない）
- 変化後は自由（濃色・透明・異物色など）
- ただし、発光色は禁止（内部光は鈍い光のみ）

---

### ● プレビュー内の色
- プレビューは“静かな器”のため、背景色は無彩色
- 宝石の色変化に応じて光の色がわずかに変わる
- 過剰な色変化は禁止（世界観が壊れるため）

---

### ● 色の禁止事項
- 強い彩度の UI パーツ（宝石以外）は禁止
- 発光色（ネオン・蛍光）は禁止
- UI 全体に色を敷くことは禁止
- 白金属に色を混ぜることは禁止

---

### ● 色ルールの目的
- UI の静けさを守る  
- 工場の空気層を壊さない  
- 宝石（つまみ）だけが“色の主役”になるようにする  

## ■ UI の空気層（透明レイヤー）仕様

RefineRoom の UI は「工場の空気層」を壊さないため、
透明レイヤーの扱いを厳密に管理する。

---

### ● 空気層の役割
- UI と背景（工場）を分離する“薄い膜”
- 影を弱め、光を柔らかくする
- UI の動きを静かに見せるための緩衝材

---

### ● 空気層の透明度
- 基本透明度：0.0〜0.3
- 最大でも 0.6 を超えない
- 色は持たない（完全無彩色）
- 濁りは禁止（世界観が壊れるため）

---

### ● 空気層の厚み
- 厚みは視覚化しない（見えない）
- ただし UI の動きに合わせて“わずかに屈折”する
- 屈折は 0.1 秒以内に収束する

---

### ● 空気層と UI パーツの関係
- 白金属パーツは空気層の上に置かれる
- 宝石つまみは空気層をわずかに揺らす（内部光と連動）
- プレビューは空気層の奥に置かれる（奥行きの演出）

---

### ● 空気層の禁止事項
- 色をつけることは禁止
- 影を落とすことは禁止
- 過剰な屈折・揺れは禁止
- UI 全体を覆う“膜のような表現”は禁止（重くなるため）

---

### ● 空気層の目的
- UI の静けさを守る  
- 工場の背景を壊さない  
- 宝石（つまみ）の存在感を引き立てる  

## ■ 工場の動き（Factory Motion）

RefineRoom の背景にある工場は、実際に“動いている”様子が視覚化される。  
ただし UI を邪魔しないよう、動きは奥行きのあるレイヤーで管理される。

---

### ● 工場の基本状態（常時稼働）
- 遠景に巨大な機械がゆっくり回転している
- ベルトコンベアが低速で動いている
- 圧延機・ローラー・アームなどが周期的に動く
- 動きはゆっくりだが、確実に“作っている”感じがある

---

### ● 工場の動きの種類（視覚化される）
- **回転**：巨大な輪・歯車・ローラーがゆっくり回る  
- **往復**：アームやプレス機が一定周期で動く  
- **流れ**：素材がベルト上をゆっくり流れる  
- **光の走り**：加工ラインに沿って白金の光がスッと走る  
- **粒子の反応**：空気中の微細な粒が機械の動きに反応して揺れる  

※ すべて “静かで重い工場” の動き

---

### ● UI との関係（ここが重要）
- UI を操作すると、工場の動きが“わずかに強まる”  
  - つまみを動かす → 加工ラインの光が少し強くなる  
  - レバーを引く → 遠景の機械が少しだけ加速する  
- ただし UI が揺れることはない（空気層が緩衝材）

---

### ● 工場の光のルール
- 色は持たない（白金属の反射光のみ）
- 点滅ではなく“流れる光”
- 強すぎる光は禁止（UI が負けるため）
- 光は機械の動きに同期する

---

### ● 工場の音の扱い
- 音は“完全無音”  
- ただし視覚的な動きで“重さ”を感じる  
（あなたの世界観の静けさを守るため）

---

### ● 工場の目的
- UI の背後で“実際に作られている”世界を示す  
- 宝石（つまみ）の変化が、工場の動きとつながっていることを示す  
- GRUMA 全体の世界観に“生命感”を与える  

## ■ 編集工程と工場の連動（Process Sync）

RefineRoom では、ユーザーが編集画面で行う操作が、
背後の工場の動きと直接つながっている。  
UI は“工場の制御卓”として機能し、操作に応じて工場が反応する。

---

### ● ① スライダー（宝石つまみ）を動かす
- 工場の加工ラインの光が強くなる
- 遠景のローラーがわずかに加速する
- 粒子が加工ラインに沿って流れるように動く
- プレビューの変化と工場の動きが同期する

→ **宝石の調整＝工場の微調整**

---

### ● ② レバーを引いて UI を展開する
- 工場の奥で大型アームがゆっくり動き出す
- 加工ラインの光が“起動”のように走る
- 遠景の機械が低速稼働から“作業モード”に入る

→ **レバー＝工場のモード切り替え**

---

### ● ③ 素材パネルで項目を選ぶ
- 選んだ項目に応じて工場の一部が動く  
  - 色 → 染色ラインが動く  
  - 光 → 反射調整ラインが動く  
  - 形 → 成形ラインが動く  
- 遠景の機械の動きが項目に合わせて変化する

→ **選択＝工場の工程切り替え**

---

### ● ④ プレビューが変わる
- 工場のどこかで“加工が完了した”ような光が走る
- 粒子がふわっと舞い上がる
- 遠景の機械が一瞬だけ強く動き、すぐ静まる

→ **プレビュー更新＝工場の出力**

---

### ● ⑤ 保存（ジュエリー化）
- 工場の奥で“完成ライン”が光る
- 遠景のアームが完成品を運ぶような動きをする
- 粒子が静かに収束し、空気層が落ち着く

→ **保存＝工場が作品を送り出す**

---

### ● 連動の原則
- UI の操作は工場の“動きの強さ”に影響する  
- 工場は UI を揺らさない（空気層が緩衝材）  
- 工場の動きは UI の“裏側の真実”として存在する  
- すべての動きは静かで重く、過剰にならない

細長い白金属レバー

■ 操作
レバーを倒すと、工場奥の光反射板が同じ角度にスッと動く

微調整すると光の揺れが変わる

手ごたえは “軽いけど精密”

プレビューの光の揺れがリアルタイムで変化

● ③ 色タンク UI（Color Tank）
役割：色の深さ・濁り・透明度の調整

回転速度で “色の深さ” が変わる

粘性のある重さの操作感

■ 工場の反応
色タンクの液体がゆっくり揺れる

宝石内部の光が変化

プレビューの色が静かに変わる

● ④ 微粒子ライン UI（Particle Line）

■ 工場の機械UI（Factory Machine UI）

RefineRoom の右パネルには、工程に応じて以下の 5 つの工場機械UIが切り替わって表示される。
ユーザーは中央プレビューを見ながら、右側の機械を直接操作する。

1. 成形ローラー UI（形）
2. 光反射板 UI（光）
3. 色タンク UI（色）
4. 微粒子ライン UI（質感）
5. プレスアーム UI（仕上げ）

各機械UIは、操作に応じて工場の対応ラインがリアルタイムで動き、
プレビューに結果が反映される。
