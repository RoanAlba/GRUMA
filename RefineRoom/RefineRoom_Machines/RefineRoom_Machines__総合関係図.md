# RefineRoom_Machines 総合関係図

RefineRoom は GRUMA の「操作卓・魂の部屋」であり、  
RefineRoom_Machines はその内部に存在する「巨大機構・身体の領域」である。

両者は常に循環しながら作品を生成し、  
“意図 → 加工 → 表示 → 調整” のループによって作品は成長する。

本ファイルは、RefineRoom と RefineRoom_Machines の  
世界観・工程・データ循環を統合した総合関係図である。

---

## 1. 全体構造（Overview）

RefineRoom（操作・意図）
　↓ 指示・数値・素材
RefineRoom_Machines（内部機構・加工）
　↓ 結果・形状・光・色・質感
RefineRoom（表示・確認）

この循環によって作品は “生まれ、育ち、整えられる”。

---

## 2. RefineRoom の役割（操作卓）

- 素材選択  
- 調整レバー（形・光・色・質感）  
- 図面モード  
- 保存レバー  
- 出品レバー  
- プレビュー表示  
- 呼吸アニメーション  
- 儀式具の操作  

RefineRoom は **「操作」と「表示」** を担う。  
ここでの意図・調整・儀式が、内部機構へ送られる。

---

## 3. RefineRoom_Machines の役割（内部機構）

- 成形ライン  
- 光反射ライン  
- 色処理ライン  
- 微粒子ライン  
- 仕上げライン  
- 保存処理  
- 出品処理  
- 学習システム  

RefineRoom_Machines は **「計算・加工・統合・儀式」** を担う。  
RefineRoom の操作を実体化し、作品の身体を生成する領域。

---

## 4. データの流れ（Data Flow）

● ① 素材投入  
RefineRoom → RefineRoom_Machines（成形ライン）

● ② 工程処理  
成形 → 光反射 → 色処理 → 微粒子 → 仕上げ

● ③ 結果返却  
RefineRoom_Machines → RefineRoom（プレビュー更新）

● ④ 調整  
RefineRoom のレバー操作 → RefineRoom_Machines に再送 → 再計算

● ⑤ 保存  
RefineRoom（保存レバー）  
→ RefineRoom_Machines（保存処理）  
→ UI System（カード生成）

● ⑥ 出品  
RefineRoom（出品レバー）  
→ RefineRoom_Machines（最終儀式）  
→ 完了カード生成  
→ RefineRoom に戻る

---

## 5. 図面モードとの関係（Blueprint Link）

図面モードは RefineRoom_Machines の内部構造を線図化して表示する。

RefineRoom（図面モード）  
　↓  
RefineRoom_Machines（形状・寸法・内部構造データ）  
　↓  
RefineRoom（線図表示）

図面モード中は光・色・質感が弱まり、線図が最優先で表示される。

---

## 6. 学習システムとの関係（Learning Link）

RefineRoom の操作ログ  
　↓  
RefineRoom_Machines（学習システム）  
　↓  
工程ラインの最適化  
　↓  
RefineRoom のプレビューが自然に良くなる

学習は RefineRoom_Machines の内部で行われる。

---

## 7. 儀式具との関係（Save / Publish）

● 保存レバー  
RefineRoom → RefineRoom_Machines（保存処理）  
→ UI System（カード生成）  
→ RefineRoom（+2% 明るいプレビュー）

● 出品レバー  
RefineRoom → RefineRoom_Machines（最終儀式）  
→ 完了カード生成  
→ 画面暗転（25%）  
→ 深い呼吸（+4% → 0%）

保存＝状態の刻印  
出品＝不可逆の儀式
