# RefineRoom プレビュー仕様
プレビューは RefineRoom の 中心の器。
素材・光・質感・調整結果を 静かに受け取り、表示するための領域。

目的は
「工房の空気を壊さず、結果だけを静かに見せる」  
という世界観を UI として成立させること。

1. プレビューの役割（Role）
素材の精製結果を表示する

調整レバーの変化を即時反映する

工房の“静かな中心”として存在する

UI System の情報を持ち込まない（素材専用）

呼吸アニメーションで工房の生命感を表現する

2. プレビューの形状（Shape）
● 形
角丸なしの矩形

枠線なし

影なし

“器” ではなく “空気層” として存在する

● 背景
工房の空気色（薄い灰白）

大理石の気配は入れない

反射は弱い

● サイズ
画面中央の 40〜48%

左右のパネルとバランスを取る

大きすぎず、小さすぎず

3. プレビューの配置（Placement）

左：素材パネル　｜　中央：プレビュー　｜　右：調整レバー

画面中央に固定

素材スロットは左下に寄せる

レバー類（保存・出品）は右下に寄せる
→ プレビューは 視線の中心 に来るように設計

4. プレビューの状態（States）
● ① 空（Empty）
背景のみ

呼吸アニメーションは弱め

素材スロットが空のときの状態

● ② 素材が読み込まれている（Active）
素材の形・色・質感が表示

呼吸アニメーションが通常強度

調整レバーと連動して変化

● ③ 完了状態（Completed）
呼吸が浅くなる

光の反射が弱くなる

編集不可

UIカードの完了状態と一致

5. プレビューのアニメーション（Animation）
● 呼吸（Breathing）
明るさ：+2% → 0%

時間：2.0〜2.5秒

ループ：ゆっくり

工房の生命感を表現

● 素材が変わった瞬間
明るさ：+3% → 0%（0.2秒）

真鍮は光らない

バウンドなし

音なし

● 調整レバー操作時
変化は即時反映

アニメーションは最小限

“動いているのに動いていないように見える” が正解

6. プレビューと他UIの連動（Link）
● 素材スロット
素材が置かれた瞬間、プレビューが息を吸う

素材の初期値が反映される

● 調整レバー
数値・光量・質感が即時反映

呼吸アニメーションは維持

● 保存レバー
保存時にプレビューが +2% 明るくなる（0.2秒）

● 出品レバー
出品時は呼吸が一度だけ深くなる

完了状態に移行

7. プレビューの禁止事項（NG）
枠線

角丸

強い影

派手な光

バウンド

回転

素材の切り替え時の派手なアニメーション

プレビューを小さくする

プレビューをポップアップ化する

プレビューに UI System の情報を表示する

→ プレビューは “静かな器” であり、情報を詰め込む場所ではない。

8. 目的（Design Purpose）
プレビューは
「工房の中心にある静かな空気の器」  
として存在する。

枠がない

影がない

呼吸する

素材だけが静かに浮かぶ

これによって、
RefineRoom の世界観が 視覚的にも精神的にも安定する。

✔ 最短まとめ
プレビューは 工房の中心の器

枠なし・影なし・角丸なし

呼吸アニメーションで生命感

素材スロット・調整レバー・レバー類と連動

完了状態では呼吸が浅くなる

RefineRoom フォルダに保存する
