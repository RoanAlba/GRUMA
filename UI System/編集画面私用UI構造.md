
# 編集画面と私用UIの構造
GRUMA世界の UI における「編集画面」と「私用UI」の役割と構造を定義する。
目的は 素材を自由に試し、気に入ったものだけを“自分のUI”として使える仕組み を作ること。

## 1. 基本思想（Concept）
UI は 素材スロット方式 で構成される。

編集画面＝すべての素材が並ぶ「倉庫」

私用UI＝選んだ素材だけを配置した「自分専用UI」

この2つは役割が明確に分かれており、
モード切り替えは存在しない。

## 2. 編集画面（Edit Screen）
● 役割
UI 素材を“試す場所”

すべてのパーツが並ぶ

形状・素材・色・レール・つまみなどを自由に組み合わせられる

ここで作ったものを「保存」すると私用UIに反映される

● 配置される要素
スライダー（真鍮レール＋陶器つまみ）

レバー（小・大）

ボタン

パネル素材（灰白・大理石・木材など）

装飾ライン（真鍮）

背景素材（窓の景色など）

● 特徴
UI画像はすべてここに置く

使わない素材もここに残しておく

倉庫としての役割を持つため“全部入り”で良い

## 3. 私用UI（My UI）
● 役割
編集画面で気に入ったものだけを配置した“自分専用UI”

実際の操作に使うUI

シンプルで、必要なものだけが並ぶ

● 特徴
編集画面とは別の場所

保存した素材だけが並ぶ

UIの“本番環境”

余計な素材は一切置かない

## 4. 保存・呼び出し・出品（Save / Load / Export）
● 保存（小レバー）
編集画面で作ったUIセットを保存

私用UIに反映される

「自分用にする」行為

● 呼び出し（ボタン）
保存したUIセットを呼び出す

過去のセットを切り替えられる

● 出品（大レバー）
完成したUIセットを“売る用”として出力

モード切り替えは不要

行為として独立している

## 5. 編集画面 → 私用UI の流れ
編集画面で素材を組み合わせる

気に入ったら「保存（小レバー）」

私用UIに反映される

必要なら「呼び出し（ボタン）」で切り替え

完成したら「出品（大レバー）」で出力

→ モード切り替えは存在しない。
　行為だけが存在する。

## 6. UI画像の扱い（Image Handling）
編集画面には“全部のUI画像”を置く

私用UIには“選んだものだけ”を置く

使わない素材は編集画面に残しておく

画像の置き場所に迷わない構造

## 7. 禁止事項（NG Rules）
編集画面と私用UIを混ぜる

モード切り替えを作る

保存と出品を同じ操作にする

編集画面に“私用UI専用素材”を置かない

私用UIに“全部入り”を置く

## 8. 目的（Design Purpose）
この構造の目的は
「試す場所」と「使う場所」を分けることで、
UI制作を直感的でストレスなくすること。

✔ 最短まとめ
編集画面＝素材倉庫

私用UI＝選んだものだけ

保存＝小レバー

呼び出し＝ボタン

出品＝大レバー

モード切り替えは不要

UI画像は編集画面に全部置く
