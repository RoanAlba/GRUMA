# UIカードのサムネ内容仕様
UIカードに表示するサムネイルの内容を定義する。
目的は “一目で UI セットの雰囲気と構造がわかる” こと。
情報量は最小限にし、GRUMA の静けさを保つ。

## 1. サムネの役割（Role）
UI セットの「雰囲気」を伝える

スライダー数・色味・構造の“気配”を示す

詳細を詰め込みすぎない

一覧パネルで迷わないための“記号”として機能する

## 2. 表示する要素（Elements）
サムネに入れる情報は 3つだけ。

● ① スライダーの本数（最大3本まで）
表示は 最大3本

4本以上ある場合は「3本＋気配の影」で表現

真鍮レール＋陶器つまみのミニチュア版

位置は上部に寄せる

→ UIセットの“構造の特徴”が一目でわかる。

● ② パネルの色味（背景）
灰白・大理石・黒革など

実際のパネル色を 薄く・控えめに 表現

質感は 20〜30% の弱い表現（主張しすぎない）

→ 世界観の統一を保ちながら、セットの個性を示す。

● ③ 主要アクセント（1つだけ）
真鍮ライン

レバーの影

装飾の“気配”

つまみの色（黒 or 白）

※ 1つだけに限定
→ 情報量を増やさず、セットの“象徴”だけを見せる。

## 3. 表示しない要素（Not Included）
サムネは“情報を削る”のが正解。

テキスト

数値

レバーの詳細形状

ボタンの種類

細かい装飾

背景の部屋

影を濃くすること

実寸の UI をそのまま縮小すること

→ 情報過多になると一覧パネルがうるさくなる。

## 4. レイアウト（Layout）
サムネはカード上部に配置

横幅の 60%

高さはカードの 35〜40%

影は 1%（気配程度）

角丸なし（カードの形と統一）

## 5. 色と質感（Color & Texture）
全体の彩度は低め（20〜30%）

真鍮は控えめな光沢

パネル色は実物より少し薄く

つまみは黒 or 白のどちらか

→ 世界観の“静けさ”を守る。

## 6. サムネの生成ルール（Rules）
● ルール1：情報は「3つまで」
スライダー

パネル色

アクセント1つ

● ルール2：実物をそのまま縮小しない
あくまで“象徴”として描く

● ルール3：迷わないことが最優先
似た UI セットでも、サムネで区別できるようにする

## 7. 禁止事項（NG Rules）
情報を詰め込みすぎる

スライダーを4本以上描く

背景を濃くする

真鍮をギラつかせる

文字を入れる

実寸の UI をそのまま縮小する

カードの世界観を壊す色を使う

## 8. 目的（Design Purpose）
UIカードのサムネは
「UIセットの象徴を静かに示す小さな儀式札」  
として存在する。

情報を削り、
世界観を壊さず、
一覧パネルで迷わないための“記号”として機能する。

✔ 最短まとめ
サムネは スライダー3本・パネル色・アクセント1つ

情報は最小限

実物を縮小しない

世界観の静けさを守る

UI System フォルダに保存する
