# UI System README
GRUMA の UI を構成する 共通UI・操作体系・儀式具・一覧パネル など、
UI 全体の仕組みを統合的にまとめたドキュメント。

目的は 「UI System の全体像を一目で理解できるようにする」 こと。

1. UI System の役割
UI System は GRUMA の UI を支える 共通ルールの集合。

UI の世界観（静けさ・儀式性）を統一する

各 UI の動き・光・質感を揃える

誤操作を防ぎ、迷わない操作体系を作る

RefineRoom や他の部屋でも共通して使える基盤を提供する

2. UI System の構成
UI System は以下の要素で構成される。

一覧パネル（保存UIの管理）

UIカード（サムネ・情報欄・状態）

レバー類（保存・出品）

共通アニメーション

共通レイアウトルール

禁止事項（世界観を壊す要素）

3. 一覧パネル（List Panel）
保存した UI セットを管理する中心機能。

● 構成
レイアウト

ヘッダー

カード配置

アニメーション

閉じ方

総合仕様（親ファイル）

● 目的
「静かに中央に現れ、静かに選び、静かに消える儀式の場」

● 仕様ファイル
一覧パネル_総合仕様

一覧パネル_レイアウト仕様

一覧パネル_ヘッダー仕様

一覧パネル_カード配置仕様

一覧パネル_アニメーション仕様

一覧パネル_閉じ方仕様

4. UIカード（UI Card）
UIセットを象徴する “儀式札”。

● 構成
サムネ

情報欄

ホバー

クリック

選択状態

削除

並び替え禁止

● 目的
「静かに息づく儀式札」 として UI セットを識別する。

● 仕様ファイル
UIカード_サムネ内容仕様

UIカード_情報欄仕様

UIカード_ホバー仕様

UIカード_クリック動作仕様

UIカード_選択状態仕様

UIカード_削除動作仕様

UIカード_並び替え禁止仕様

5. レバー（儀式具）
UI System の中でも特に重要な “決断の装置”。

● 種類
保存レバー（軽い決意）

出品レバー（最終決断）

● 目的
「操作を儀式化する」  
軽い操作ではなく、決断の重みを身体で感じるための装置。

● 仕様ファイル
出品レバー仕様

6. 共通アニメーション
UI System 全体で統一される “動きのルール”。

● 原則
動きは短く、静か

光は真鍮だけ

バウンド禁止

音なし

0.1〜0.2秒の儀式的な動き

7. 共通レイアウトルール
角丸なし

真鍮ラインは細く

余白は広め

背景は灰白（青み5%）

情報量は最小限

UIは“静けさ”を最優先

8. 禁止事項（NG）
UI System 全体で禁止される要素。

派手なアニメーション

音

バウンド

情報過多

角丸

強い色

並び替え

ポップアップ乱発

UIの縮小表示（実寸をそのまま小さくしない）

9. UI System の目的（Design Purpose）
UI System は
「GRUMA の世界観を UI に落とし込むための基盤」。

静けさ

儀式性

真鍮の光

大理石の気配

余白の美学

誤操作のない操作体系

これらをすべて統一し、
どの画面でも “GRUMA の空気” を感じられるようにする。
