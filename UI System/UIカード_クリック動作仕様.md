# UIカードのクリック動作仕様
UIカードをクリック（タップ）したときの 反応・状態遷移・動き を定義する。
目的は 静けさ・儀式性・誤操作防止 を両立した選択動作を作ること。

## 1. クリックの役割（Role）
UIセットを 選択状態にする

選択された UIセットを 編集画面に読み込む

完了状態のカードは選択不可

情報を増やさず、静かに反応する

## 2. クリック時の反応（Reaction）
クリックした瞬間に起きる反応は 3つだけ。

● ① 真鍮縁が光る
明るさ：+15%

時間：0.2秒

イージング：ease-out
→ 儀式具としての“反応”。

● ② カードが 1px 浮く
時間：0.1秒

選択状態の証として使う
→ ホバーとの差別化。

● ③ 四隅の淡い光が点灯
サイズ：1〜2px

明るさ：弱い

時間：0.15秒
→ 選択状態の継続を示す。

## 3. 状態遷移（State Transition）
クリックによって状態がどう変わるか。

● 未選択 → 選択
真鍮縁が光る

1px 浮く

四隅の光が点灯

情報欄の明るさが +2%（気配程度）

● 選択 → 未選択
四隅の光が消える（0.2秒）

浮きが戻る

明るさが元に戻る

● 完了 → 選択（不可）
完了状態のカードはクリックしても反応しない

真鍮縁も光らない
→ “もう触らなくていい” 空気を守る。

## 4. クリック後の動作（After Click）
● 選択された UIセットを読み込む
編集画面に遷移

選択状態は一覧パネルを閉じるまで保持

一覧パネルを閉じたら選択状態はリセット

● 読み込みアニメーション
画面が 0.15秒だけ暗くなる

UIセットが編集画面に展開される

音は出さない

## 5. 禁止事項（NG Rules）
クリックでカードを大きくする

情報をポップアップ表示する

枠線を太くする

色を変えて状態を示す

強い光や点滅

音を鳴らす

ダブルクリックを必要にする

完了カードを選択可能にする

→ 世界観の“静けさ”と“儀式性”を守るため。

## 6. ホバーとの関係（Relation）
ホバー：影1px濃くなる＋真鍮3%明るくなる

クリック：真鍮が光る＋1px浮く＋四隅の光

完了：光なし・呼吸浅い

→ 3つの階層が自然に区別できる。

## 7. 目的（Design Purpose）
UIカードのクリック動作は
「儀式札に触れたときの静かな決意」  
として存在する。

光・浮き・呼吸の変化で
世界観を壊さずに “選んだ” を伝える。

✔ 最短まとめ
クリック＝真鍮が光る＋1px浮く＋四隅の光

完了カードは反応しない

選択状態は一覧パネルを閉じるまで保持

UI System に保存する
