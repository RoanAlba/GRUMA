## 1. 情報欄の役割（Role）
UIセットを識別するための“静かなラベル”

情報を詰め込まない

サムネの補助として機能する

一覧パネルの世界観を壊さない

## 2. 表示する情報（Elements）
情報欄に入れるのは 2つだけ。

● ① UIセット名
自動生成名（例：Set 01 / Set 02）

またはユーザーが付けた名前

フォント：細めサンセリフ

色：濃い灰（黒ではない）

サイズ：14〜15px

● ② 保存日時
表記例：2026/05/18 14:32

色：薄い灰（主張しない）

サイズ：11〜12px

UIセット名の下に小さく配置

## 3. レイアウト（Layout）
コード
[ UIセット名 ]
[ 保存日時 ]
左寄せ

上下余白：8〜12px

カード下部の 60〜65% の領域に配置

行間は狭め（静けさを保つ）

## 4. 情報欄の背景（Background）
カード背景と同じ

境界線なし

影なし

情報欄だけ色を変えない（世界観が壊れるため）

## 5. 禁止事項（NG Rules）
アイコンを入れる

テキストを増やす（タグ・説明文など）

色を変える

枠線を付ける

情報欄だけ背景を変える

日時を大きく表示する

文字を中央揃えにする

## 6. 目的（Design Purpose）
UIカードの情報欄は
「静かに UI セットを識別するための小さな札」  
として存在する。

情報は最小限、
世界観はそのまま、
迷わず選べるようにするための補助。

✔ 最短まとめ
情報欄は必要

UIセット名＋保存日時だけ

左寄せ・控えめ・静か

UI System に保存する
