# 一覧パネルのカード配置仕様
一覧パネル内に表示される UIカードの 並び・余白・列数・スクロール構造 を定義する。
目的は 静けさ・視認性・迷わなさ を両立した配置を作ること。

## 1. 基本レイアウト（Layout）
一覧パネルのカードは 縦スクロールの 1 列配置。

列数：1 列固定

カード幅：パネル幅の 88〜92%

カード間の縦余白：20〜28px

左右余白：24〜32px

→ “棚に静かに並んでいる” 印象を作る。

## 2. カードの並び順（Order）
● 並び順のルール
新しい順（上に新しい）

保存レバーで保存した瞬間、最上段に追加

完了状態のカードは下に寄せる（自動）

● 並び替え機能は付けない
世界観の静けさを守るため

操作を増やさないため

## 3. カードのサイズ（Size）
高さ：120〜140px

幅：パネル幅の 88〜92%

角丸：なし（一覧パネルの直線的世界観に合わせる）

影：1%（気配程度）

## 4. カードの内部構造（Internal Layout）
カード内部は 上：サムネ / 下：情報 の 2 段構造。

コード
[ サムネ（上部 35〜40%） ]
[ 情報（下部 60〜65%） ]
● サムネ
横幅 60%

上部に寄せる

情報量は最小限（別仕様参照）

● 情報
UIセット名（自動生成 or ユーザー名）

保存日時（控えめな灰色）

完了状態のアイコンは置かない（光で示す）

## 5. スクロール構造（Scroll）
縦スクロールのみ

慣性は弱め

バウンドなし

スクロールバーは細い真鍮ライン（透明度 60%）

→ “棚をゆっくり見ていく” 感覚。

## 6. カードの選択状態との関係（Relation）
選択状態は 光・浮き で表現（別仕様参照）

配置は変わらない

選択してもカードの位置は動かない

完了状態は下に寄せる（自動）

## 7. カードの最大表示数（Limit）
一覧パネル内に表示するカード数は 無制限

ただし、スクロールが長くなりすぎないよう
20件を超えると“区切りの薄い真鍮ライン”を挿入

→ 長い一覧でも迷わない。

## 8. 禁止事項（NG Rules）
2列以上にする

カードを小さくしすぎる

情報を詰め込みすぎる

カードの色を変える

並び替え機能を付ける

スクロールを速くする

バウンドアニメーション

→ 世界観の“静けさ”を守るため。

## 9. 目的（Design Purpose）
一覧パネルのカード配置は
「静かに棚に並ぶ儀式札」  
として存在する。

1列・広めの余白・弱い影によって
GRUMA の世界観を壊さず、
ユーザーが迷わず UIセットを選べるようにする。

✔ 最短まとめ
1列固定

カード幅 88〜92%

縦余白 20〜28px

新しい順

サムネ上・情報下

スクロールは弱い慣性

UI System フォルダに保存する
