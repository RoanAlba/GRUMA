# UIカードの削除動作仕様
UIカードを削除する際の 操作方法・確認・アニメーション・誤操作防止 を定義する。
目的は 静けさを保ちながら、絶対に誤操作を起こさない削除動作 を作ること。

## 1. 削除の役割（Role）
UIセットを 完全に消去する

誤操作を防ぐため、削除は “儀式的で重い操作” とする

保存や出品とは完全に別の行為

## 2. 削除の入口（Entry）
削除は カードの右上にある小さな真鍮の点（Delete Dot） から行う。

サイズ：6〜7px

色：真鍮（暗め）

ホバー：明るさ +3%（控えめ）

クリック：削除確認モーダルへ遷移

→ ボタンではなく “点” にすることで、誤操作を防ぐ。

## 3. 削除確認モーダル（Confirmation Modal）
削除は 必ず確認モーダルを挟む。

● モーダル内容
タイトル：「このUIを削除しますか」

説明：「この操作は取り消せません」

ボタン：

削除（真鍮の細い枠）

キャンセル（灰白）

● モーダルの動き
出現：95% → 100%（0.18秒）

真鍮縁が 0.1秒だけ光る

背景暗幕：40%

## 4. 削除実行時の動き（Delete Action）
削除ボタンを押した瞬間：

● ① カードがゆっくり暗くなる
明るさ：100% → 0%（0.25秒）

● ② 真鍮縁が光らずに消える
削除は儀式ではないため光らせない

● ③ カードが縮む
スケール：100% → 92%（0.2秒）

● ④ カードが霧散するように消える
透明度：100% → 0%（0.25秒）

● ⑤ 一覧が詰めて再配置される
アニメーション：0.15秒

静かに詰まる（バウンドなし）

## 5. 削除後の状態（After Delete）
カードは完全に消える

選択状態はリセット

スクロール位置は保持

完了カードの位置は維持

## 6. 禁止事項（NG Rules）
削除をワンタップで実行

削除ボタンを大きくする

派手なアニメーション

音を鳴らす

削除後にポップアップを出す

カードを赤くする（世界観崩壊）

## 7. 目的（Design Purpose）
UIカードの削除動作は
「静かに、確実に、誤操作なく消える儀式」  
として存在する。

削除は軽く扱わず、
世界観の静けさを保ちながら
“本当に消す” という重みを演出する。

✔ 最短まとめ
削除は必要

カード右上の小さな真鍮点から開始

必ず確認モーダル

削除時は光らず、静かに霧散

UI System に保存する
