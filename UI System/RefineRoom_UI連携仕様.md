# RefineRoom UI連携仕様
一覧パネルで選択した UIセットを RefineRoom（編集画面）に読み込むための仕組み を定義する。
目的は 一覧 → 編集 の流れを儀式的かつ静かに成立させること。

1. 全体フロー（Overview）

一覧パネルを開く
　↓
UIカードをクリック（選択）
　↓
選択状態が確定
　↓
RefineRoom に UIセットが読み込まれる
　↓
一覧パネルが静かに閉じる

この流れを 一貫した儀式 として扱う。

2. UIカード選択 → 読み込みの条件
● 読み込みが発生するのは 選択状態になった瞬間
真鍮が光る

1px 浮く

四隅の光が点灯
→ この “選択儀式” が完了したら読み込み開始。

● 完了カードは読み込めない
完了状態は “固定”

クリックしても反応しない

読み込みも発生しない

3. 読み込みアニメーション（Loading Ritual）
RefineRoom に UIセットを読み込むときは、
短い儀式的な暗転 を挟む。

● 動き
画面全体が 0.15秒だけ暗くなる（20%）

RefineRoom の UI が “息を吸うように” 1%だけ明るさが変わる

音はなし

バウンドなし

真鍮は光らない（読み込みは儀式ではないため）

● 目的
“切り替わった” を静かに伝える

でも派手にしない

世界観の空気を壊さない

4. 読み込まれる内容（Data Mapping）
UIカードに保存されている内容は、RefineRoom の以下の要素に反映される。

● ① スライダー位置
すべてのスライダーが保存時の位置に戻る

反映は 一瞬で（アニメーションなし）

● ② 素材・光・質感の設定
RefineRoom のプレビューに即時反映

反映後のプレビューは “静かに息をする”（呼吸アニメーション）

● ③ 選択中の項目
左パネルの選択項目も復元

ただし UI System の項目は復元しない（素材だけ）

● ④ 保存レバーの状態
保存レバーは “通常位置” に戻る

レバーの位置は保存しない（儀式具だから）

5. 一覧パネルの閉じ方（連携時）
UIカードを選択して読み込みが完了したら、
一覧パネルは自動で閉じる。

● 動き
0.15秒でフェードアウト

スケール：100% → 97%

暗幕：40% → 0%

真鍮は光らない（出現時のみ）

● 目的
“選んだら閉じる” を自然にする

ユーザーに余計な操作をさせない

儀式の流れを途切れさせない

6. 読み込み後の状態（After Load）
RefineRoom は選択された UIセットの状態で完全に復元

スライダーはすぐ操作可能

保存レバーもすぐ使える

一覧パネルを再度開くと、選択状態はリセットされている

7. 禁止事項（NG Rules）
読み込み中のローディングアイコン

読み込み音

派手な暗転

スライダーのアニメーション復元

一覧パネルを閉じずに読み込む

読み込み完了ポップアップ

完了カードを読み込めるようにする

→ 世界観の静けさと儀式性を守るため。

8. 目的（Design Purpose）
RefineRoom への UI連携は
「儀式札を選び、その力を工房に呼び戻す」  
という世界観の動作。

選択は儀式

読み込みは静かな暗転

パネルは自然に閉じる

工房は選んだ状態に息を吹き返す

この流れが GRUMA の UI体験を支える。

✔ 最短まとめ
UIカード選択 → 即読み込み

読み込み時は0.15秒の静かな暗転

完了カードは読み込めない

読み込み後に一覧パネルは自動で閉じる

RefineRoom は保存時の状態に完全復元

UI System に保存する
