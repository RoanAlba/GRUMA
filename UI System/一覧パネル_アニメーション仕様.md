# 一覧パネルのアニメーション仕様
保存した UI セットを一覧表示するパネルの 出現・消失・内部動作 のアニメーション仕様を定義する。
目的は 静けさ・儀式性・迷わなさ を両立した動きを作ること。

## 1. 出現アニメーション（Appear）
一覧パネルが画面中央に現れるときの動き。

● 動き
中央に向かって“ふわっと集まる”

スケール：

95% → 100%（0.18秒）

透明度：

0% → 100%（0.18秒）

イージング：ease-out（柔らかく止まる）

● 背景暗幕
透明度：0% → 40%（0.18秒）

色：黒に近い灰（青み 5%）

● 真鍮縁の反応
パネルの真鍮縁が 0.12秒だけ細く光る

光はすぐに消える（残光なし）

→ 儀式具が“召喚される”ような静かな出現。

## 2. 消失アニメーション（Disappear）
一覧パネルを閉じるときの動き。

● 動き
スケール：

100% → 97%（0.15秒）

透明度：

100% → 0%（0.15秒）

イージング：ease-in（静かに消える）

● 背景暗幕
40% → 0%（0.15秒）

● 真鍮縁
光らせない（出現時だけの儀式）

→ “静かに霧散する”ような消え方。

## 3. 内部スクロール（Scroll）
一覧パネル内のカードをスクロールするときの動き。

● スクロール挙動
慣性：弱め（距離短め）

速度：ゆっくり

バウンド：なし（儀式感を壊すため禁止）

● スクロールバー
細い真鍮ライン

透明度：60%

動き：上下に滑らかに追従

→ “棚をゆっくり見ていく”感覚。

## 4. カードのホバー・タップ（Card Interaction）
● ホバー（PC）
影が 1px だけ濃くなる

真鍮縁がわずかに明るくなる（0.1秒）

● タップ（選択）
真鍮縁が 0.2秒だけ光る

カードが 1px 浮く

すぐ元に戻る

→ 儀式具の“反応”として統一。

## 5. 呼び出しボタンとの連動（Link）
呼び出しボタンを押す → 一覧パネルが出現

一覧パネルを閉じる → 呼び出しボタンは元の状態

ボタンの光とパネルの光が 重ならないように 設計

## 6. 禁止事項（NG Rules）
バウンドアニメーション

強い拡大縮小

派手なフェード

音を鳴らす

真鍮縁を長く光らせる

スクロールを速くする

→ 世界観の“静けさ”を守るため。

## 7. 目的（Design Purpose）
一覧パネルのアニメーションは
「静かに現れ、静かに消える儀式」  
として存在する。

光・動き・速度のすべてが
GRUMA の世界観に沿うように設計されている。

✔ 最短まとめ
出現：95%→100%、0.18秒、真鍮が一瞬光る

消失：100%→97%、0.15秒、光らない

スクロール：弱い慣性、真鍮バー

カード：タップで真鍮が光る

UI System フォルダに保存する
