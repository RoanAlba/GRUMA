# 呼び出しボタンの仕様
保存した UI セットを一覧パネルとして呼び出すためのボタンの仕様を定義する。
目的は 静かで儀式的な操作感 を保ちながら、
ユーザーが迷わず一覧パネルを開けるようにすること。

## 1. ボタンの役割（Role）
保存した UI セットを呼び出す

一覧パネルを開く唯一の入口

編集画面・私用UIのどちらからでも使える

モード切り替えではなく「行為」として存在する

## 2. ボタンの形状（Shape）
形：真鍮の薄い円盤

直径：22〜26mm

厚み：1.5〜2mm（出っ張り感を抑える）

角：丸め処理

表面：セミグロス（ギラつき禁止）

→ 儀式具の“押し札”のような静けさ。

## 3. ボタンの配置（Placement）
編集画面：右上 or 左上に固定

私用UI：右上に固定

他の UI 要素と干渉しない位置

画面端からの余白：24〜32px

→ 常に「ここにある」とわかる位置。

## 4. ボタンのアイコン（Icon）
形：三本線（≒メニュー）ではなく、真鍮の“輪”

中央に小さな点（保存UIの“核”を示す）

線の太さ：1.2mm

色：柔らかい金（黄60% / 赤40%）

→ 世界観に合う“呼び出しの印”。

## 5. ボタンの反応（Interaction）
● タップ
真鍮の輪が 0.15秒だけふわっと光る

一覧パネルが中央に展開される

音は出さない

● 長押し
反応なし（設定メニューなどは存在しない）

→ 操作は「押す」だけに限定。

## 6. 一覧パネルとの連動（Behavior）
ボタンを押す → 一覧パネルが中央に出現

一覧パネルを閉じると、ボタンは元の状態に戻る

ボタンは一覧パネルの背後に隠れない（常に見える）

→ UI System の“入口”としての一貫性。

## 7. 禁止事項（NG Rules）
三本線アイコン（世界観に合わない）

厚みのあるボタン（出っ張り感が強い）

強い光沢（安っぽくなる）

ボタンに文字を入れる（情報量が増える）

押したときに音を鳴らす（静けさが壊れる）

## 8. 目的（Design Purpose）
呼び出しボタンは
「静かに UI を呼び出すための儀式具」  
として存在する。

真鍮の輪・薄さ・控えめな光で、
GRUMA の世界観を壊さずに一覧パネルへ導く。

✔ 最短まとめ
真鍮の薄い円盤

中央に小さな点のアイコン

押すと真鍮がふわっと光る

一覧パネルが中央に展開

出っ張りゼロ・音ゼロ・情報量最小

世界観に完全一致
