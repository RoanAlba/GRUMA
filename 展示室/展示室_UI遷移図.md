# 展示室 UI 遷移図（正式版 / 保存用）

## 1. 概要
本書は、展示室における UI の遷移構造を定義する。  
展示室は「本棚UI → 作品詳細UI → 本棚UI」という  
静かで儀式的な往復動線を持つ。

---

## 2. 遷移図（全体構造）

                ┌──────────────────────┐
                │      展示室（入口）      │
                │   ・壁は固定             │
                │   ・光は柔らかい間接光   │
                └─────────┬────────┘
                              │
                              ▼
                ┌──────────────────────┐
                │      本棚UI（一覧）     │
                │  ・作品カードが横に並ぶ │
                │  ・灰磁小物が隙間に置かれる│
                │  ・横スクロールで歩行感 │
                └─────────┬────────┘
                              │ クリック
                              ▼
                ┌──────────────────────┐
                │   作品詳細UI（儀式空間） │
                │ ・作品画像を大きく中央表示│
                │ ・背景は白磁のまま       │
                │ ・閉じるボタンは真鍮     │
                └─────────┬────────┘
                              │ フェードアウト
                              ▼
                ┌──────────────────────┐
                │   本棚UI（元の位置に戻る）│
                │ ・歩行感を維持            │
                └──────────────────────┘

---

## 3. 遷移の性質（重要ポイント）

### 3-1. 展示室 → 本棚UI
- 自動遷移ではなく、展示室のメイン画面として本棚UIが表示される  
- 壁は動かず、本棚UIだけが横スクロールする

### 3-2. 本棚UI → 作品詳細UI
- 作品カードをクリック  
- 遷移は **0.25〜0.35秒のフェードイン**  
- 背景は少し暗くなる（黒10〜15%）  
- 作品画像が“ふわっと”浮かぶ

### 3-3. 作品詳細UI → 本棚UI
- 閉じるボタン（真鍮）をクリック  
- **フェードアウトで戻る**  
- 本棚UIは **元のスクロール位置に戻る**（歩行感を維持）

---

## 4. 光と影の遷移上の扱い

### 4-1. 本棚UI
- 光源は固定  
- 影は薄い  
- 小物とカードの影だけが動く（歩行感）

### 4-2. 作品詳細UI
- 光は少し強めに受けるが眩しくない  
- 影はほぼ消える（儀式性のため）  
- 背景は白磁のまま

---

## 5. 禁止事項（遷移に関するもの）
- 派手なアニメーション  
- スライド遷移  
- 拡大アニメーション  
- 背景を動かす  
- 作品ごとに光を変える  

---

## 6. 目的と効果（総括）
- 展示室の動線を「静か・儀式的・余白のある体験」に統一  
- 本棚UIの歩行感を壊さず、詳細UIへ自然に遷移  
- 工房 → 展示室 → 詳細 の世界観を完全に接続  
- UI ではなく “空間としての展示室” を成立させる
