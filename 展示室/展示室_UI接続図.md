# 展示室 UI 接続図（正式版 / 保存用）

## 1. 概要
本書は、展示室における  
「本棚UI」「作品詳細UI」「作品カード」「展示室全体構造」  
がどのように接続され、  
ユーザーがどの順番で閲覧するかを定義する。

展示室は “静かな鑑賞空間” として設計されており、  
UIは極力控えめに、空間に溶け込むように存在する。

---

## 2. 全体接続図（俯瞰）

                ┌──────────────────────┐
                │     本棚UI（入口）      │
                │ ・作品一覧               │
                │ ・カテゴリ選択           │

                              ▼

                └──────────────────────┘

## 3. 各UIの役割

### 3-1. 本棚UI（入口）
- 展示室の入り口  
- 作品の一覧を静かに並べる  
- カテゴリ選択が可能  
- UIは“棚の一部”として存在する

### 3-2. 作品詳細UI
- 作品を大きく見せる鑑賞画面  
- 説明文・タグ・光の演出  
- UIは極薄で、作品が主役  
- スクロールはゆっくり（速い動きは禁止）

### 3-3. 作品カード（展示物）
- 展示室に置かれる “作品の実体”  
- UIではなく “物体” として扱う  
- 作品詳細UIから生成される  
- 展示室の光を受けて存在感が出る

### 3-4. 展示室全体構造
- 展示室の空間そのもの  
- 部屋の配置、動線、光の流れ  
- UIではなく “建築” の領域

---

## 4. 遷移の性質（重要）

### 4-1. 本棚UI → 作品詳細UI
- 作品を選ぶと静かにフェードイン  
- 遷移は0.25秒以内  
- 背景は展示室の光を保ったまま

### 4-2. 作品詳細UI → 作品カード
- 「展示する」操作で生成  
- 作品カードは展示室の床に配置される  
- UIではなく “物体” として扱われる

### 4-3. 作品カード → 展示室全体構造
- 展示室の空間に自然に溶け込む  
- UI遷移ではなく “空間の一部になる”

---

## 5. 光と影の扱い（接続時）

### 5-1. 本棚UI
- 光：柔らかい白磁  
- 影：極薄  
- UIは棚に溶け込む

### 5-2. 作品詳細UI
- 作品を照らす光が主役  
- UIは光を邪魔しない透明感

### 5-3. 作品カード
- 展示室の光を受けて存在感が出る  
- UIではなく “展示物” として扱う

---

## 6. 禁止事項
- 派手なアニメーション  
- スライド遷移  
- 不透明な背景  
- 強い影  
- カラーライト  
- 作品ごとに色を変えること  
- UIが作品より目立つこと

---

## 7. 目的と効果（総括）
- 展示室のUIと空間を統合  
- 鑑賞の流れを儀式的に整理  
- UIではなく “作品と空間” が主役になる  
- 世界観（白磁・灰磁・真鍮）と完全に一致させる
